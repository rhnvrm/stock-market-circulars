## Summary

BSE has scheduled a mock trading session for the Equity segment on Saturday, January 17, 2026. The circular provides disaster recovery (DR) site connection parameters for trading members to connect to the BOLT Plus trading system. Members can connect using BOLT TWS, third-party trading applications from empanelled vendors, or in-house developed systems through ETI APIs. The circular includes technical connection details for both primary and secondary DR sites, as well as weblinks for various exchange web-based systems.

## Key Points

- Mock trading session scheduled for Saturday, January 17, 2026, for Equity segment
- Trading members can connect to DR site of BOLT Plus using BOLT TWS or third-party applications
- Connection available through ETI APIs for in-house developed systems
- Primary DR connection: IP 10.255.241.10, Port 18909
- Secondary DR connection: IP 10.255.241.11, Port 18909
- BOLT TWS users need to change technical connection parameters in configuration settings window
- Access available via Shift+F12 shortcut keys or scrip profile icon
- Web-based systems (RTRMS, CLASS collateral, Extranet, etc.) accessible through provided URLs

## Regulatory Changes

No regulatory changes introduced. This is a routine technical and operational circular for mock trading preparation.

## Compliance Requirements

- Trading members must configure their systems with DR site connection parameters
- BOLT TWS users must update technical connection parameters in configuration settings
- Third-party application users must update ETI API connection parameters with provided IP addresses and ports
- Members using online trade file application must change remote server IP and port settings in setting.ini file
- Existing user IDs and passwords can be used for web-based system access

## Important Dates

- **January 17, 2026 (Saturday)**: Mock trading session for Equity segment

## Impact Assessment

**Market Impact**: None. This is a mock trading session for testing purposes with no impact on actual trading or market operations.

**Operational Impact**: Low. Trading members need to perform technical configuration changes to participate in the mock session. This is a routine exercise to ensure disaster recovery preparedness and system connectivity. The mock session allows members to test their DR connectivity without affecting live trading operations.

**Member Impact**: Trading members who wish to participate in the mock session need to update their system configurations with the provided DR site parameters. Participation is voluntary and serves as a technical readiness check.
