## Summary

BSE has scheduled a mock trading session for the Equity segment on Saturday, January 17, 2026. The circular provides technical connectivity parameters for trading members to connect to the Disaster Recovery (DR) site of the BOLT Plus trading system. Trading members can connect using BOLT TWS, third-party trading applications, or in-house developed systems through ETI APIs.

## Key Points

- Mock trading session scheduled for Saturday, January 17, 2026, for Equity segment
- Trading members can connect to DR site using BOLT TWS or third-party applications through ETI APIs
- DR connectivity parameters provided for Primary DR (IP: 10.255.241.10, Port: 18909) and Secondary DR (IP: 10.255.241.11, Port: 18909)
- Access details provided for web-based systems including RTRMS, CLASS collateral, EBOSS, LEIPS, and Extranet
- Configuration changes required in BOLT TWS using Shift+F12 or scrip profile icon
- Trading members need to update technical connection parameters for the mock session

## Regulatory Changes

No regulatory changes introduced. This is a technical testing circular.

## Compliance Requirements

- Trading members must configure their systems with DR site connection parameters
- BOLT TWS users need to change technical connection parameters in configuration settings window
- Third-party trading application users must update ETI API connection parameters with specified DR IP addresses and ports
- Members connecting to web-based systems should use provided URLs with existing credentials

## Important Dates

- **January 17, 2026 (Saturday)**: Mock trading session for Equity segment

## Impact Assessment

**Market Impact**: None. This is a mock trading exercise with no impact on actual market operations.

**Operational Impact**: Low. Trading members need to temporarily reconfigure their systems to connect to the DR site for testing purposes. This is a routine exercise to ensure business continuity preparedness.

**Member Impact**: Minimal. Voluntary participation in mock trading to test DR connectivity and system readiness.
