## Summary

BSE has scheduled a mock trading session for the Commodity Derivatives segment on Saturday, December 13, 2025. This session is designed to test disaster recovery (DR) site connectivity and ensure business continuity preparedness. Trading members can connect to the DR site using BOLT TWS or third-party trading applications through ETI APIs with specified technical parameters.

## Key Points

- Mock trading session scheduled for Saturday, December 13, 2025
- Applies to Commodity Derivatives segment only
- DR site connectivity testing for BOLT Plus trading system
- Multiple connection methods available: BOLT TWS and third-party ETI API applications
- Technical connection parameters provided for Primary and Secondary DR sites
- Web-based systems (RTRMS, CLASS, Extranet, etc.) accessible through specified URLs

## Technical Connection Parameters

**For BOLT TWS Users:**
- Change technical connection parameters in configuration settings window at login
- Access via scrip profile icon or "Shift+F12" shortcut keys

**For Third-Party Trading Applications (ETI APIs):**
- Primary DR: IP 10.255.241.26, Port 14910
- Secondary DR: IP 10.255.241.27, Port 14910

**Web-Based Systems Access:**
- RTRMS: https://rtrms.bseindia.com
- CLASS Collateral: https://classseg.bseindia.com
- EBOSS: https://eboss.bseindia.com
- Extranet: https://member.bseindia.com
- iBBS: https://ibbs.bseindia.com

## Compliance Requirements

- Trading members must configure their systems with DR site connection parameters
- Members using BOLT TWS need to update configuration settings
- Members using third-party applications must change technical parameters in their trading applications
- Existing user IDs and passwords can be used for web-based systems

## Important Dates

- **Mock Trading Date:** Saturday, December 13, 2025

## Impact Assessment

This is a routine business continuity and disaster recovery testing exercise with minimal operational impact. No regular trading operations are affected as the mock session is scheduled on a Saturday. Trading members should use this opportunity to verify their DR site connectivity and ensure readiness for any contingency scenarios. The exercise helps maintain market infrastructure resilience and operational preparedness.
