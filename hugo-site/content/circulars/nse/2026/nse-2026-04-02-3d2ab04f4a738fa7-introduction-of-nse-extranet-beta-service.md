## Summary

NSE is introducing a new Extranet service called **NSE Extranet Beta** as part of its ongoing efforts to enhance trading member user experience. This new platform will coexist with the existing Extranet service and provides file-based information exchange between trading members and the Exchange via the integrated Member Portal.

## Key Points

- All members with existing Extranet access will automatically have access to NSE Extranet Beta.
- Existing sub-user IDs with current Extranet access will be granted access to NSE Extranet Beta by default.
- New sub-user IDs must be configured by the Member Admin using the Full Member Portal Admin User Guide Ver 2.0.
- Reports will be available on NSE Extranet Beta 30 minutes after publication on the existing Extranet.
- The platform is accessible via the Member Portal at https://inspection.nseindia.com/MemberPortal/.
- Authentication requires User ID, Member code, Password, CAPTCHA, and OTP validation.
- Best viewed on Internet Explorer 11.0 and above or Google Chrome (preferred).

## Regulatory Changes

No regulatory or compliance rule changes are introduced. This is an operational technology enhancement — a parallel Extranet platform to the existing service.

## Compliance Requirements

- Member Admins must assign the following three roles to sub-users for NSE Extranet Beta access:
  - **NSE Extranet Beta** — Mandatory
  - **Extranet - common** — Mandatory
  - **Extranet - dnld** — Mandatory
- New sub-users requiring access must follow the steps in the attached User Manual (Full Member Portal Admin User Guide Ver 2.0).
- No action is required for existing sub-users, as they will be granted access by default.

## Important Dates

- **Circular Date:** April 02, 2026
- No specific go-live or deadline date mentioned in the circular.

## Impact Assessment

This update is operationally significant for all NSE trading members. The introduction of NSE Extranet Beta runs in parallel with the existing Extranet, ensuring uninterrupted access to reports and services during transition. Members should note the 30-minute delay in report availability on the new platform versus the existing one. Member Admins will need to take action only for new sub-user creation; existing users are migrated automatically. No market, trading, or compliance impact is anticipated.
